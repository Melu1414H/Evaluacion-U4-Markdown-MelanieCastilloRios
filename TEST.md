# Evaluación de Conocimientos: GitHub y Markdown

## 1. Conceptos Generales
**¿Qué es un archivo Markdown y para qué sirve?**
> Es un documento como texto plano que se añade como: ".md" para que sea más claro y ordenado en el cual se utilizan caracteres para poder dar un buen formato de negrillas, titulos, subtitulos o listas. 
 
**¿Cuál es el propósito del archivo README.md en un repositorio de GitHub?**
*El propósito de este es brindar información para las personas que empiezan con este tema o programa de como iniciar recien en GitHub.*

## 2. Investigación de Sintaxis Markdown
Completa la siguiente guía de referencia con los símbolos correctos:
 
| Elemento | Sintaxis Markdown (Ejemplo) | Resultado Visual |
|----------|-----------------------------|------------------|
| Título Nivel 1 | `# Texto` | # Texto |
| Título Nivel 2 | `## Texto` | ## Texto |
| Texto en Negrita | `**Texto**` | **Texto** |
| Texto en Cursiva | `*Texto*` | *Texto* |
| Lista Desordenada | `- Elemento` | • Elemento |
| Lista Ordenada | `1. Elemento` | 1. Elemento |
| Insertar un Enlace | `[Nombre](https://google.com)` | [Enlace](https://google.com) |

## 3. Flujo de Trabajo en GitHub Desktop
**¿Qué diferencia hay entre hacer un "Commit" y hacer un "Push"?**
* **Commit:** Es para guardar los cambios en tu computadora, es decir que no se suben a internet todavía.
* **Push:** Es enviar los cambios que hiciste desde tu computadora hacia GitHub para que estén en línea.
 
**¿Para qué sirve el botón "Fetch Origin"?**
> Sirve para actualizar la información del repositorio y ver si hay cambios nuevos en GitHub sin descargarlos todavía.
